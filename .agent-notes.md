# Agent Notes

## 2026-06-27 — Knockout resolution and projected fixtures

- `/api/scores` production issue was a Vercel function module-load crash from an extensionless ESM import in `api/scores.ts`; the API build gate now runs `tsc -p tsconfig.api.json --noEmit`.
- Round-of-32 fixture labels are intentionally static in `fixtures.json`; runtime display resolves them from the live scores feed.
- `src/data/bracket.ts` is the canonical frontend resolver for bracket and schedule display:
  - `1A` / `2B` style slots resolve from current group standings.
  - `3A/B/...` slots resolve through `src/data/thirdPlaceAllocation.ts`.
  - `projectedKnockoutTeams` is true when a visible matchup still depends on unfinished groups or unstable third-place allocation.
- Projected fixtures use a compact `TriangleAlert` icon with `aria-label` / tooltip text from `asItStands`; no visible text chip.
- `/api/share` has a small resolver for `/match/:id` metadata. It reuses `computeStandings` and the third-place allocation helpers; do not import frontend-only modules into Vercel functions unless `tsconfig.api.json` proves the import is API-safe.
- Regression tests added:
  - `src/data/bracket.test.ts` covers resolved group slots, projected current standings, Germany vs Paraguay, and Argentina vs Cape Verde.
  - `src/data/thirdPlaceAllocation.test.ts` guards table shape, the current `ABDEFGIL` allocation row, and stability detection.
