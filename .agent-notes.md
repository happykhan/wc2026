# Agent Notes

## 2026-06-27 — Knockout resolution and projected fixtures

- `/api/scores` production issue was a Vercel function module-load crash from an extensionless ESM import in `api/scores.ts`; the API build gate now runs `tsc -p tsconfig.api.json --noEmit`.
- Round-of-32 fixture labels are intentionally static in `fixtures.json`; runtime display resolves them from the live scores feed.
- `src/data/knockoutSlots.ts` is the shared API-safe resolver for bracket, schedule, and `/match/:id` share metadata:
  - `1A` / `2B` style slots resolve from current group standings.
  - `3A/B/...` slots resolve through `src/data/thirdPlaceAllocation.ts`.
  - each slot resolves to `placeholder`, `projected`, or `final`.
- `src/data/bracket.ts` adapts that resolver into `BracketTeam`; `projectedKnockoutTeams` is true when a visible matchup still depends on unfinished groups or unstable third-place allocation.
- Projected fixtures use a compact `TriangleAlert` icon with `aria-label` / tooltip text from `asItStands`; no visible text chip.
- `/api/share` reuses the shared knockout slot resolver for `/match/:id` metadata; do not import frontend-only modules into Vercel functions unless `tsconfig.api.json` proves the import is API-safe.
- Regression tests added:
  - `src/data/bracket.test.ts` covers resolved group slots, projected current standings, Germany vs Paraguay, and Argentina vs Cape Verde.
  - `src/data/thirdPlaceAllocation.test.ts` guards table shape, the current `ABDEFGIL` allocation row, and stability detection.
  - `src/data/knockoutSlots.test.ts` guards the shared resolver and projected-to-final transition.

## 2026-06-27 — Bracket mobile views

- Bracket page now has two display modes:
  - `Rounds`: mobile-first round tabs with vertical match cards.
  - `Tree`: the original horizontally scrollable full bracket.
- The mode is stored in `localStorage` as `wc2026-bracket-view`; first visit defaults to `Rounds` on mobile and `Tree` on desktop.
- `BracketMatch` carries `sourceTeam1` / `sourceTeam2` so the UI can trace `W73` / `L101` dependencies without reverse-engineering from display labels.
- Path view opens from a team row route icon and shows the winner path through downstream matches. It is display-only; no prediction state is stored.

## 2026-06-27 — Feature notices

- `src/components/FeatureNotice.tsx` renders dismissible, versioned feature notices backed by `localStorage` keys under `wc2026-seen-feature-*`.
- `FeatureNoticeGroup` shows one unseen notice at a time so multiple announcements never overlap.
